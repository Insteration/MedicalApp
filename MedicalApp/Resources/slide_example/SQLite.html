SQLite обычная таблица и таблица для FTS

Обновить

December 2018

Просмотры

851 раз

1

У меня есть таблица новости (идентификатор, news_id, news_title) и я FTS таблицу создающий:

CREATE VIRTUAL TABLE news_search USING fts4 (news_title, tokenize=porter);

Я использую триггер, чтобы сохранить таблицу NEWS и news_search в синхронизации:

CREATE TRIGGER IF NOT EXISTS insert_news_trigger 
AFTER INSERT ON news 
BEGIN 
INSERT OR IGNORE INTO news_search (news_title) VALUES (NEW.news_title);
END;

Вопрос: как пользоваться поиском? Когда я MATCH в news_search таблице он возвращает мне только записи из этой таблицы, но мне нужно * news_id * из новостей таблицы. Может быть , я должен добавить * news_id * столбец news_search таблицы?

Что такое правильный способ использовать FTS в SQLite?

sqlite full-text-search fts3 fts4
Magog

1 ответы

3

Read the documentation; FTS tables also have a rowid column (also called docid) that you can set explicitly to the same value as the corresponding key of the original table.

Assuming that news.id is the rowid (i.e., INTEGER PRIMARY KEY), you should change your trigger to also copy that ID value into the news_search table. You can the use that to look up the original record:

SELECT *
FROM news
WHERE id IN (SELECT docid
             FROM news_search
             WHERE news_title MATCH '😸')
